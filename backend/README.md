# TeleAssist — RAG Telecom Customer Care Chatbot v2.0

A production-grade **Retrieval-Augmented Generation** chatbot for telecom support featuring:
- **Source citations** in every answer (FAQ, Ticket ID, Guide page number)
- **Confidence gate** — skips the LLM entirely when no relevant sources are found
- **FastAPI** backend with real-time SSE streaming
- **React** frontend with streaming bubbles and a collapsible citations panel

---

## Architecture

```
User question  (React → POST /chat)
      │
      ▼
FastAPI  api.py
      │
      ▼
rag_chain.ask()
      │
      ├── retriever.py: retrieve_with_scores()
      │     ├── ChromaDB · faq      top-3 with L2 scores
      │     ├── ChromaDB · tickets  top-3 with L2 scores
      │     └── ChromaDB · guides   top-3 with L2 scores
      │
      ├── Confidence gate (threshold = 1.0 L2 distance)
      │     └── all scores > threshold → SSE {"type":"fallback"}
      │
      ├── _format_docs_with_citations()  — context block with [citation] headers
      │
      └── ChatPromptTemplate → Qwen3-32B (Groq) → StrOutputParser
            │  (streams tokens)
            ▼
      SSE events → React (sources panel + streaming bubble)
```

## Setup

### Backend

```bash
cd backend
uv sync                    # or: pip install -e .
cp .env.example .env       # fill in GROQ_API_KEY and HF_TOKEN

# Ingest data once
python ingest_faq.py
python ingest_tickets.py
python ingest_pdf.py

# Start API server
uvicorn api:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev                # http://localhost:5173
```

## How Citations Work

Each retrieved chunk gets a citation string assigned at retrieval time:

| Source | Example citation |
|---|---|
| FAQ (faq.csv) | `FAQ #2 [data]` |
| Resolved ticket (tickets.db) | `Ticket TK-007 [connectivity]` |
| PDF guide (telecom_guide.pdf) | `Guide p.3 chunk 4` |

These strings are injected into the LLM system prompt. The model is instructed to append the citation tag after every claim. The UI renders a collapsible **Sources** panel below each assistant message, with relevance bars showing the similarity score.

## Confidence Gate

`retrieve_with_scores()` uses `similarity_search_with_score()` (L2 distance). Any chunk with distance above `SIMILARITY_THRESHOLD = 1.0` is filtered out. If **all** chunks are filtered, the LLM is skipped and the fallback message is returned instantly:

> "I'm sorry, I don't have enough information in my knowledge base to answer your question confidently. Please **call 611** or open the **MyTelecom app** for personalised support."

## Key Tech

- **LangChain** — orchestration, prompts, output parsing
- **ChromaDB** — vector store (3 collections: faq, tickets, guides)
- **sentence-transformers/all-MiniLM-L6-v2** — local embedding model
- **Qwen3-32B on Groq** — LLM (fast inference, free tier available)
- **FastAPI + uvicorn** — async API server with SSE
- **React + Vite** — frontend (no heavy component libraries)
